# Programming Basics

In our journey to becoming AI-first software engineers, understanding the core building blocks of programming is essential. Every complex system is built upon a foundation of data and logic.

## Primitive vs. Non-Primitive Data Types

One of the first distinctions to master is the difference between individual (primitive) values and collections (non-primitive) of data.

### 1. Primitive Data Types
These are the most basic forms of data that represent a single value.

| Type | Description | Example |
| --- | --- | --- |
| **Integer** | Whole numbers without decimals. | `10`, `-5` |
| **Float** | Numbers with fractional parts (decimals). | `3.14`, `0.001` |
| **String** | Sequences of characters (text). | `"Hello"`, `'Python'` |
| **Boolean** | Logical values representing Truth or Falsehood. | `True`, `False` |

### 2. Non-Primitive Data Types
Also known as **Data Structures**, these types can store multiple values or collections of data in a structured way.

*   **Lists**: Ordered, mutable collections (e.g., `[1, 2, 3]`).
*   **Tuples**: Ordered, immutable collections (e.g., `(1, 2, 3)`).
*   **Sets**: Unordered collections of unique elements (e.g., `{1, 2, 3}`).
*   **Dictionaries**: Key-value pairs (e.g., `{"name": "Alice"}`).

---

## Mutability: A Key Concept

*   **Mutable**: The data can be changed after creation (Lists, Dictionaries, Sets).
*   **Immutable**: The data cannot be changed once created (Integers, Strings, Tuples).

Understanding mutability is crucial for memory management and avoiding side effects when passing data between functions.
